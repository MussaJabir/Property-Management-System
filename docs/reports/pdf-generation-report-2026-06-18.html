Production Fix Report
Date: 18 June 2026
• PDF document generation

PDF document generation is now fully working on the live system. Landlords and renters can download official receipts, invoices, lease contracts and reports as clean, professional PDF files — directly from the live site, on phone or computer.

This had previously been switched off while the system ran on a small starter server. With the server now upgraded, we have restored it, given it the resources it needs to run reliably, and changed the release process so it stays in place permanently.

What we changed

1. Restored PDF document generation on the live system Fix
What we found: when PMS was first put online, document generation was deliberately turned off because the small starter server did not have enough memory to run it. After the server’s memory was upgraded, two things on the live server still needed attention before PDFs could be produced: the production server was missing the engine that turns a document into a PDF, and the amount of memory set aside for the application was too low to run that engine.
  • The PDF engine is now installed on the production server, so the system can render documents into finished PDF files.
  • It now applies to every document type: receipts, invoices, lease contracts and exported reports.
  • Confirmed on the live site — a real receipt was generated and downloaded successfully end to end.
What it means: the paperwork landlords and renters actually need — proof of payment, bills, signed contracts, financial reports — can now be produced and shared as proper documents straight from the system.
2. Gave it the resources to run reliably Performance
Producing a PDF is a heavier task than a normal screen, so we increased the application’s memory allowance on the upgraded server. This lets a document render comfortably without slowing down or interrupting the rest of the system.
  • The memory available to the application was raised so PDF rendering has the headroom it needs.
  • Tested under the real production setup, not just in development — the document generated cleanly.
What it means: downloads are dependable rather than hit-or-miss, and the rest of the system keeps running smoothly while a document is being prepared.
3. Made the fix permanent and self-sustaining Reliability
We also improved the way updates are released so this configuration is kept in place automatically on every future update. Previously, server settings could drift out of step with the master copy and a routine update could quietly undo a fix. That gap is now closed.
  • The live server’s configuration now always matches the official master copy.
  • Future improvements to settings roll out automatically with each release — no manual adjustments on the server.
  • This particular fix — and others like it — can no longer be accidentally reversed by a later update.
What it means: a problem we solve once stays solved. The system becomes steadier and cheaper to maintain over time, with fewer surprises after updates.
